2. INSPECTION LOGIC & CLASSIFICATION
------------------------------------------------------------------------------

- Suspect cells flagged in annotated image, saved to logs, shown in UI
- Added _resolve_ic(): confidence-weighted retry resolution
  (w = 0.7 * conf_second + 0.3 * conf_first; PASS only if w >= 0.90)

------------------------------------------------------------------------------
3. TEMPLATE MATCHING
------------------------------------------------------------------------------
- Replaced pixel-threshold preprocessing with Canny edge detection       (4bbe06c)
- Changed IC template extraction to pin-area-only patch
  (bottom 50% of IC height)                                               (61151e1)
- Added template preview save on Confirm                                  (36f8c1a)
- Experiment: Sauvola adaptive binarization tested as alternative
  (branch: Sauvola_binary); concluded Otsu + bilateral is better;
  Sauvola removed                                                         (36f8c1a)

------------------------------------------------------------------------------
4. CAMERA & HARDWARE
------------------------------------------------------------------------------
- Fixed exposure param: ExposureTimeAbs (GigE) -> try/except fallback
  to ExposureTime (USB3)                                                  (34d7fb4)
- Added live preview via QTimer (100 ms / 10 fps) in STANDBY state;
  stops during inspection, restarts on return to standby                  (8fc87ea)
- CAMERA_FPS key sets AcquisitionFrameRateEnable + AcquisitionFrameRate
  on camera open (handles both USB3 and GigE register names)              (8893902)

------------------------------------------------------------------------------
5. OCR API INTEGRATION
------------------------------------------------------------------------------
- Added _ocr_api_call(): sends cropped IC image to external OCR API      (d7e53b9)
- Added Lot Number and Operator ID fields with validation guard           (2935cf2)
- Guards: empty-field popup, skip API when DEBUG=True, block field
  edits during run                                                        (fae3130)
- Fixed: field lock state, cancellation logging bug, empty API response   (8463cc4)
- Mark comparison now uses ocr_mark from ReadMark API (server-side)

- CreateRecord and log_ocr record actual mark used (_ocr_used_mark)
- expected_mark field kept in UI as temporary testing override
